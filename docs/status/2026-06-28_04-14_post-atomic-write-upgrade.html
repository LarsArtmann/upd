04Totally Fucked Up

Blocking

CI injects version into a non-existent symbol

.github/workflows/ci.yml:25

The CI build command is:

go build -ldflags="-s -w -X github.com/LarsArtmann/upd/internal.ProgramVersion=ci" -trimpath ./cmd/upd

There is no internal package and no internal.ProgramVersion symbol. The real variable is github.com/LarsArtmann/upd.ProgramVersion. Go silently ignores the -X assignment to a missing symbol, so CI binaries report upd/dev instead of upd/ci.

Fix Replace internal.ProgramVersion with ProgramVersion in the CI ldflags string. Validate by running the produced binary with --version.
Blocking

Three pre-existing makezero linter failures

diff.go:40, diff.go:42, render.go:167

golangci-lint reports that slices are allocated with make(..., len) but immediately indexed without initializing each element. This is a real style/strictness issue that blocks a fully clean lint run. 

// diff.go:40
lcs := make([][]int, oldLen+1)

// diff.go:42
lcs[rowIdx] = make([]int, newLen+1)

// render.go:167
segments := make([]string, len(widths))
Fix Either initialize every element explicitly, switch to append-only construction, or add targeted //nolint:makezero annotations with rationale.
